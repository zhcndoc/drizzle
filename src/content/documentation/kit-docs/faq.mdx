# 常见问题与故障排除

### **我应该使用 `generate` 还是 `push`？**

这两个命令在逻辑上是不同的。`generate` 用于创建一个与 `drizzle-kit`（或任何其他迁移工具）所需的附加信息一起的 SQL 文件。

生成这些迁移后，它们不会应用到数据库中。您需要在下一步中执行此操作。您可以在**[这里](/kit-docs/faq#how-do-i-apply-migrations-generated-by-drizzle-kit)**阅读更多相关信息。

另一方面，`push` 不需要生成任何迁移。它只是将您的模式与数据库模式同步。在使用时请务必小心，我们建议仅在本地开发和本地数据库中使用它。要了解更多信息，请参阅**[`drizzle-kit push`](/kit-docs/overview#prototyping-with-db-push)**

## 故障排除

### **为什么 drizzle-kit 无法读取我的 CLI 选项？**

Drizzle Kit 对于读取配置选项有一组当前的优先级：

1. 如果存在 `drizzle.config` 文件，则从中读取。
2. 如果不存在 `drizzle.config` 文件，则检查 CLI 中的 `--config` 选项。
3. 如果没有找到 `--config` 选项，则使用 CLI 选项；否则，从定义的配置中读取。

如果同时指定配置参数和 CLI 选项，将抛出错误，并且您应该使用 CLI 选项或在项目中定义配置文件。
